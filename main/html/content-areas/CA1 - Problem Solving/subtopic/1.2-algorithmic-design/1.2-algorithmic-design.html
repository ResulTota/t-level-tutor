1.2 - Algorithmic Design


1.2.1 - Know the definition and understand the characteristics and purpose of algorithms

Algorithms are step-by-step procedures for solving problems. They are essential in computer science and programming, providing a clear set of instructions to achieve a specific goal.

Characteristics:

  • Finite - Must finish after a certain number of steps. (Infinite Loops are not considered algorithms!)
  • Clear - Each step must be precise and understandable.
  • Effective - Should be able to solve the problem in a reasonable time and correctly.
  • Input - Can take zero or more inputs.
  • Output - Produces one or more outputs.
  • Logical Sequence - Follows a Structured Order.

Purpose:

  • To solve provlems efficiently.
  • To provide a clear and repeatable process.
  • To communicate logic between developers, and stakeholders.
  • To allow problems to be broken down into smaller tasks.

1.2.2 - Know and understand ways that algorithms can be expressed

Flowcharts:

Written Descriptions:

Express algorithms in clear, step-by-step plain English using hierarchical markers (numbers, letters, indentation) to show sequence, decisions, and sub-steps.

Example:

1. Start

2. Input user’s age

    2.1 IF age >= 18 THEN

        2.1.1 Output “Adult”

    2.2 ELSE

        2.2.1 Output “Child”

3. End

Code:

Algorithms can be expressed directly in program code.

Example in Python:

age = input("Enter your age: ")
if age >= 18:
    print("Adult")
else:
    print("Child")

1.2.3 - Know and understand the benefits and drawbacks of the ways of expressing algorithms in flowcharts

Benefits of Flowcharts:

  • Visual Clarity: Easy to understand and follow the logic.
  • Standardised Symbols: Provides a common language for developers.
  • Logical Flow: Clearly shows the sequence of steps and decisions.
  • Problem Breakdown: Helps in breaking down complex problems into manageable parts.

Drawbacks of Flowcharts:

  • Complexity: Can become cluttered and hard to read for large algorithms.
  • Limited Detail: May not capture all nuances of the algorithm.
  • Time-Consuming: Creating flowcharts can be time-consuming compared to writing code.
  • Not Executable: Flowcharts cannot be run as code, requiring translation into a programming language.

1.2.4 - Know and understand the benefits and drawbacks of the ways of expressing algorithms in written descriptions

Benefits of Written Descriptions:

  • Simple to Write: Easy to create quickly without special tools or symbols.
  • Accessible: Understandable by both technical and non-technical audiences.
  • Structured Clarity: Hierarchical markers (numbers, indentation) make the sequence clear.
  • Good for Documentation: Useful for recording algorithm steps in a written format.

Drawbacks of Written Descriptions:

  • Ambiguity: May be unclear or misinterpreted if not written precisely.
  • Lack of Visual Flow: Harder to see branching and looping compared to flowcharts.
  • Limited for Complexity: Becomes difficult to manage with larger algorithms.
  • Not Executable: Must be translated into code before it can run.

1.2.5 - Know and understand the benefits and drawbacks of the ways of expressing algorithms in code

Benefits of Code:

  • Executable: Can be run directly to test and verify the algorithm.
  • Precision: Removes ambiguity by following strict syntax rules.
  • Modifiable: Easy to update, optimise, and debug as requirements change.
  • Real-World Application: Necessary for implementing algorithms in actual software.

Drawbacks of Code:

  • Language-Specific: May need rewriting for different programming languages.
  • Steeper Learning Curve: Requires programming knowledge and understanding of syntax.
  • Less Accessible: Harder for non-programmers to understand compared to flowcharts or written steps.
  • Can Be Overwhelming: Beginners may struggle with debugging and reading code-heavy logic.

1.2.6 - Know and understand actions to control ordering of steps in algorithms

Key Actions:

  • Sequence: Steps are carried out in a specific order, one after the other.
    Example: Input two numbers → Add them → Output the result.
  • Selection: A decision is made and the algorithm follows one path or another (IF, ELSE).
    Example: IF age >= 18 THEN Output “Adult” ELSE Output “Child”.
  • Iteration: A set of steps is repeated until a condition is met (loops).
    Example: WHILE password is incorrect → Ask for password again.

1.2.7 - Be able to determine the purpose of an algorithm and how it works

Explanation:

This skill involves analysing an algorithm to figure out what problem it solves and how the steps achieve the solution. This can be done by tracing through each step logically.

Example:

        1. Input number
        2. Multiply number by 2
        3. Output result

Purpose: This algorithm doubles a given number.

1.2.8 - Be able to determine the output of an algorithm given an input

Explanation:

By tracing through an algorithm with a given input, you can calculate the output step by step.

Example:

        Input: 5
        Algorithm: 
        1. Multiply number by 3
        2. Subtract 2
        3. Output result

Output: (5 × 3) - 2 = 13

1.2.9 - Be able to identify errors in an algorithm

Explanation:

Errors in algorithms can be logical (wrong steps or conditions), syntax-related (in code), or runtime errors (e.g., division by zero).

Example:

        Algorithm: 
        1. Input two numbers
        2. Divide first number by second number
        3. Output result

Error: Division by zero if the second number is 0.

1.2.10 - Be able to correct errors in an algorithm

Explanation:

Once an error is identified, the algorithm must be modified to fix it while maintaining the intended purpose.

Example (Correction of previous error):

        1. Input two numbers
        2. IF second number = 0 THEN
            Output “Error: cannot divide by zero”
        ELSE
            Divide first number by second number
            Output result

1.2.11 - Be able to translate between the different notations for algorithms

Explanation:

An algorithm can be expressed in different forms: flowcharts, written descriptions, or code. Translating between them ensures the logic remains the same regardless of the representation.

Example:

Written description: IF age >= 18 THEN Output "Adult" ELSE Output "Child"

Flowchart: Decision diamond checking age, leading to "Adult" or "Child".

Code:

        if age >= 18:
            print("Adult")
        else:
            print("Child")

1.2.12 - Be able to design algorithms and solutions that use actions

Explanation:

This involves creating new algorithms using the three core actions: sequence, selection, and iteration to solve a problem effectively.

Example Design:

        Problem: Find the sum of all even numbers between 1 and 10.

        Algorithm:
        1. Start
        2. Set total = 0
        3. FOR number from 1 to 10
            IF number MOD 2 = 0 THEN
                    total = total + number
        4. Output total
        5. End

Output: The sum of even numbers = 30
