2.7 - Functions and Procedures


2.7.1 - Functions

Definition: A function is a block of code that performs a specific task and returns a value. It may take parameters as input and is used to calculate or process data, making code reusable and organized.

Characteristics:

  • May or may not take parameters.
  • Must return a result.

Purpose: Reuse code, make programs modular and readable, and perform calculations or data processing.

Example:


        def area_rectangle(length, width):
            return length * width  # returns the result

        result = area_rectangle(5, 3)
        print(result)  # Output: 15

2.7.2 - Procedures

Definition: A procedure is a block of code that performs a specific action but does not return a value. It may take parameters and is used for tasks like displaying information or updating data.

Characteristics:

  • May or may not take parameters.
  • Must not return a result.

Purpose: Perform actions rather than calculations, making code modular and organized.

Example:


        def greet_user(name):
            print(f"Hello, {name}!")  # performs action

        greet_user("Alice")  # Output: Hello, Alice!

2.7.3 - Sources of Functions and Procedures

  • User-written: Created by the programmer. Example: area_rectangle.
  • Pre-written and built-in: Provided by the programming language. Example: len("Hello").
  • Pre-written in libraries: Supplied with the language. Example: math.sqrt(16).
  • Pre-written in third-party libraries: Created by others. Example: numpy.mean([1,2,3,4]).

2.7.4 - Benefits and Drawbacks of Pre-written Code

Benefits:

  • Saves time – no need to write common functions.
  • Reduces errors – pre-tested and reliable code.
  • Encourages reusability and modular programming.

Drawbacks:

  • May not fit exact needs.
  • Can make code harder to understand if overused.
  • Dependency on external libraries can increase file size or require updates.

2.7.5 - Interpreting Code Using Functions and Procedures

When interpreting code:

  • Identify functions that return values.
  • Identify procedures that perform actions.
  • Understand what inputs are used and what outputs are produced.

Example:


        def multiply(a, b):
            return a * b

        def display_result(result):
            print("Result:", result)

        res = multiply(4, 5)
        display_result(res)  # Output: Result: 20

2.7.6 - Developing Code Using Functions and Procedures

Steps to develop code:

  • Identify repeated tasks or calculations.
  • Use a function if a value needs to be returned.
  • Use a procedure if only an action is performed.
  • Optionally use pre-written functions or libraries.

Example:


        import math  # pre-written library

        def circle_area(radius):  # function
            return math.pi * radius**2

        def print_area(radius):  # procedure
            print("Area is", circle_area(radius))

2.7.7 - Debugging Code Using Functions and Procedures

Tips for debugging:

  • Check parameters passed to functions or procedures.
  • Verify return values for functions.
  • Ensure procedures perform the intended action.
  • Use print statements or debuggers to trace execution.

Example:


        def divide(a, b):
            return a / b

        try:
            print(divide(10, 0))
        except ZeroDivisionError:
            print("Error: Cannot divide by zero!")
