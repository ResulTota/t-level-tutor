2.9.1 - Determine the logical order for actions within processes

The logical order refers to arranging steps in a process so that each action follows a sequence that makes sense and produces the correct outcome.

Purpose: Ensures tasks are performed correctly, prevents logical errors, and makes debugging easier.

Example:

  • Input length and width
  • Validate inputs
  • Calculate area
  • Display result

2.9.2 - Determine the order of operations in calculations and processes

Arrange calculations so outputs are accurate and errors are avoided.

Purpose:

  • Ensures accurate outputs
  • Prevents errors like dividing by zero or using uninitialized variables

Example:


        length = 5
        width = 3
        # Correct order: validate, then calculate
        if length > 0 and width > 0:
            area = length * width
            print("Area:", area)

2.9.3 - Determine the selection of data structures

Choosing the right data structure improves execution efficiency and memory usage.

Considerations:

  • Lists/Arrays: Fast access, sequential storage
  • Dictionaries/Hash maps: Quick lookups using keys

Example:

  • Use a list to store student scores in order
  • Use a dictionary for mapping student IDs to names for quick lookup

2.9.4 - Determine the order of actions

Arrange the sequence of tasks and operations to optimize efficiency.

Purpose:

  • Ensure fast execution
  • Minimize memory usage

Example:


        # Less efficient: sorting inside a loop
        for student in students:
            sorted_scores = sorted(scores)  # repeated unnecessarily

        # More efficient: sort once before loop
        sorted_scores = sorted(scores)
        for student in students:
            process(student, sorted_scores)

2.9.5 - Naming conventions and their purpose

Naming conventions improve readability and maintainability of code.

Types:

  • Meaningful names: Describe the purpose, e.g., total_score
  • camelCase: Words joined without spaces, first word lowercase, e.g., totalScore
  • snake_case: Words joined with underscores, e.g., total_score

Example:


        # camelCase
        studentName = "Alice"

        # snake_case
        student_name = "Alice"

2.9.6 - Impact on readability of code style conventions

Code style conventions improve clarity and readability, making programs easier to maintain.

Key considerations:

  • Naming conventions: Use meaningful names consistently
  • White space: Indent code and separate sections
  • Maximum line length: Keep lines manageable (often <80-100 characters)

Example:


        # Readable
        def calculate_area(length, width):
            return length * width

        # Less readable
        def ca(l,w): return l*w

2.9.7 - Judging the suitability of an algorithm

Evaluate how well an algorithm meets requirements and efficiency standards.

Factors to consider:

  • Efficiency in execution time
  • Efficient use of memory/storage
  • Appropriate choice of data structures
  • Correct use of data types, variables, and constants
  • Presentation: clear output format
  • Maintainability: readable and easy to modify

Example:

  • Using a dictionary for quick lookups rather than looping through a list repeatedly improves execution efficiency
  • Using descriptive variable names and comments improves maintainability
