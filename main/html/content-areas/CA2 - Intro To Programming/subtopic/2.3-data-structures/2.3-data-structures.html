2.3 - Data Structures


2.3.1 - Know the standard data structures and understand their purpose and when each is used: list, array, dictionary.

Definition & Purpose: Data structures are ways to store and organize data so it can be accessed and modified efficiently. Different types serve different purposes.

List

Definition: An ordered collection of items that can hold multiple data types.

Purpose: Store sequences of values where order matters and duplicates are allowed.

When Used: When you need to maintain an ordered collection that can grow or shrink dynamically.

Example: fruits = ["apple", "banana", "cherry"], fruits.append("orange"), print(fruits) → ["apple", "banana", "cherry", "orange"]

Benefits:

  • Dynamic size (can grow/shrink).
  • Can store multiple data types.
  • Easy to iterate and manipulate using built-in methods.

Drawbacks:

  • Slower than arrays for large numeric computations.
  • Searching elements can be slow (O(n) time complexity).
  • Uses more memory than arrays.

Array

Definition: A collection of items stored at contiguous memory locations, usually of the same data type.

Purpose: Efficiently store and access multiple values of the same type.

When Used: When performance is important for numeric computations or large datasets.

Example: import array, numbers = array.array('i', [1, 2, 3, 4]), numbers.append(5), print(numbers) → array('i', [1, 2, 3, 4, 5])

Benefits:

  • Efficient memory usage.
  • Fast access by index (O(1) time complexity).
  • Ideal for large numeric computations.

Drawbacks:

  • Fixed type (all elements must be the same).
  • Less flexible than lists.
  • Fewer built-in methods than lists.

Dictionary

Definition: An unordered collection of key-value pairs.

Purpose: Store and quickly access values based on unique keys.

When Used: When you need fast lookups or associative mapping.

Example: student = {"name": "Alice", "age": 16, "grade": "A"}, print(student["name"]) → Alice

Benefits:

  • Fast lookups and retrieval by key.
  • Keys allow easy mapping between related data.
  • Flexible for storing structured data.

Drawbacks:

  • Uses more memory than lists or arrays.
  • Keys must be immutable.
  • Unordered in versions before Python 3.7.

2.3.2 - Be able to interpret code using data structures

You must be able to read and understand code that uses lists, arrays, and dictionaries.

2.3.3 - Be able to develop code using data structures.

List

  • Add an item: .append(item)
  • Insert at a position: .insert(index, item)
  • Remove an item: .remove(item)
  • Pop last item or at index: .pop() or .pop(index)
  • Sort items: .sort()
  • Reverse items: .reverse()
  • Get length: len(list)
  • Check if item exists: item in list

Array

  • Add an item: .append(item)
  • Insert at position: .insert(index, item)
  • Remove an item: .remove(item)
  • Pop last item or at index: .pop()
  • Get length: len(array)
  • Access by index: array[index]

Dictionary

  • Add or update key-value pair: dict[key] = value
  • Access value by key: dict[key]
  • Remove key-value pair: dict.pop(key)
  • Get all keys: dict.keys()
  • Get all values: dict.values()
  • Get all items (key-value pairs): dict.items()
  • Check if key exists: key in dict

2.3.4 - Be able to debug code using data structures.

Debugging involves identifying and fixing errors when working with lists, arrays, and dictionaries.

Common Errors with Lists

  • IndexError: Accessing an element outside the list bounds.
    fruits = ["apple", "banana"]
    print(fruits[2]) # ERROR: IndexError
  • TypeError: Performing operations with incompatible types.
    numbers = [1, 2, 3]
    numbers.append("four") # Mixing types may cause logic errors
  • ValueError: Removing an item that does not exist.
    fruits.remove("cherry") # ERROR: ValueError

Common Errors with Arrays

  • TypeError: Adding a value of the wrong type.
    import array
    numbers = array.array('i', [1, 2, 3])
    numbers.append(4.5) # ERROR: TypeError
  • IndexError: Accessing an element outside the valid range.
    print(numbers[5]) # ERROR: IndexError

Common Errors with Dictionaries

  • KeyError: Accessing a key that does not exist.
    student = {"name": "Alice"}
    print(student["age"]) # ERROR: KeyError
  • TypeError: Using a mutable type as a key.
    my_dict = {[1, 2]: "value"} # ERROR: TypeError
  • Overwriting values unintentionally: Assigning a value to an existing key.
    student["name"] = "Bob" # Overwrites "Alice"

General Errors Across Data Structures

  • Mutability issues: Accidentally modifying a shared object.
    list1 = [1, 2]
    list2 = list1
    list2.append(3)
    print(list1) # Output: [1, 2, 3]
  • Incorrect iteration: Using wrong index or length, causing skipped or extra elements.
