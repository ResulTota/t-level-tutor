2.11.1 - Know and understand the algorithms for searching, how they work and when they are used: linear search, binary search.

Linear Search

Definition: A simple search algorithm that checks each element in a list one by one until the target is found or the list ends.

When Used: Works best for small or unsorted datasets.

Binary Search

Definition: A search algorithm that repeatedly divides a sorted list in half to find the target element.

When Used: Works only on sorted datasets and is more efficient than linear search for large datasets.

2.11.2 - Know and understand the algorithms for sorting, how they work and when they are used: bubble sort, insertion sort, merge sort.

Bubble Sort

Definition: A simple sorting algorithm that repeatedly compares and swaps adjacent elements until the list is sorted.

When Used: Suitable for small datasets or when simplicity is more important than efficiency.

Insertion Sort

Definition: A sorting algorithm that builds a sorted section of the list one item at a time by inserting elements into the correct position.

When Used: Efficient for small or nearly sorted datasets.

Merge Sort

Definition: A divide-and-conquer sorting algorithm that splits the dataset into halves, sorts each half, and then merges them back together.

When Used: Suitable for large datasets where efficiency is critical.

2.11.3 - Know and understand the benefits and drawbacks of using: linear search, binary search, bubble sort, insertion sort, merge sort.

Linear Search

Pros:
  • Works on both sorted and unsorted datasets.
  • Very simple to implement and understand.
  • No additional memory required.
Cons:
  • Inefficient for large datasets (O(n) time complexity).
  • Slow compared to other searching algorithms.
  • Performs unnecessary comparisons if the target is near the end or not present.

Binary Search

Pros:
  • Very efficient for large datasets (O(log n)).
  • Fewer comparisons than linear search.
  • Well-suited for repeated lookups on sorted data.
Cons:
  • Requires the dataset to be sorted beforehand.
  • More complex to implement than linear search.
  • Sorting large datasets first can reduce efficiency.

Bubble Sort

Pros:
  • Very easy to understand and implement.
  • Good for teaching sorting concepts.
  • Performs well on very small datasets.
Cons:
  • Extremely inefficient for large datasets (O(nÂ²)).
  • Performs many unnecessary comparisons and swaps.
  • Rarely used in real-world applications.

Insertion Sort

Pros:
  • Efficient for small datasets.
  • Works well on nearly sorted data.
  • Simple implementation and low memory use.
Cons:
  • Still inefficient for large datasets (O(nÂ²)).
  • Slower than more advanced sorting algorithms like merge sort.
  • Not suitable for very large datasets.

Merge Sort

Pros:
  • Very efficient with large datasets (O(n log n)).
  • Stable sort (maintains relative order of equal elements).
  • Works well with linked lists and external sorting.
Cons:
  • Requires additional memory for sublists.
  • Slower than insertion sort on very small datasets.
  • Implementation is more complex compared to bubble or insertion sort.

2.11.4 - Know and understand metrics to compare algorithms: use of memory space, execution time, number of comparisons.

Memory Space: How much memory the algorithm uses while running (e.g., merge sort needs extra memory).

Execution Time: How fast the algorithm completes (measured in time complexity).

Number of Comparisons: How many times elements are compared; affects efficiency.

2.11.5 - Know and understand best case, worst case and average case for common algorithms, using logical reasoning (Big O not required).

  • Best Case: The fastest possible scenario (e.g., searching the first element).
  • Worst Case: The slowest possible scenario (e.g., searching the last element or sorting an already reversed list).
  • Average Case: The expected performance across typical inputs.

2.11.6 - Make judgements about the suitability of using different algorithms for searching.

Linear search is suitable for small or unsorted datasets. Binary search is more efficient for large, sorted datasets.

2.11.7 - Make judgements about the suitability of using different algorithms for sorting.

Bubble sort is rarely used outside teaching. Insertion sort is good for small datasets. Merge sort is best for large datasets due to efficiency.
