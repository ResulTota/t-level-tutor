7.4 - Virtual Environments

7.4.1 - Understand the role and characteristics of common virtual environment components

Virtual environments allow multiple operating systems and applications to run independently on the same physical machine. They are widely used in cloud computing, software development, education, and enterprise systems. By separating workloads, virtualisation improves flexibility, scalability, and cost efficiency.

Virtual Machines (VMs)

A virtual machine (VM) is a software-based emulation of a physical computer. Each VM runs its own operating system (called a guest OS) and behaves as though it were a separate device, even though it shares the host machine’s hardware resources.

  • Clients:
    • Virtual PC: A complete desktop system that runs inside a VM. Useful for testing different operating systems (e.g., running Linux on a Windows machine).
    • Virtual Switch: A software switch that allows multiple VMs to communicate with each other, mimicking a physical switch in a network.
    • Virtual Router: A software router that directs traffic between virtual networks. This is often used in labs to simulate complex network topologies without physical hardware.
  • Servers: A physical server can host many virtual servers, each dedicated to different tasks (e.g., one hosting a database, another hosting a website). This is the foundation of cloud computing, where providers like AWS or Azure rent out VMs instead of physical machines.
  • Pros:
    • Resource efficiency: Multiple VMs can share the same physical hardware.
    • Isolation: A crash or virus in one VM does not affect others.
    • Flexibility: VMs can be cloned, backed up, or restored quickly.
  • Cons:
    • Performance overhead: VMs are slower than running directly on hardware.
    • Hardware demands: Requires powerful CPUs and large amounts of RAM.
    • Licensing complexity: Each VM may require its own operating system license.

Extra detail: VMs are often used to run legacy software that no longer works on modern operating systems. Developers also use them for sandboxing, where potentially unsafe code is tested in isolation.

Hypervisors

A hypervisor is the software layer that creates and manages VMs. It decides how hardware resources (CPU, memory, storage, network) are shared. Without hypervisors, virtualisation would not be possible.

Type 1 (Bare-Metal)

These run directly on the host hardware, with no underlying operating system. They are considered the most secure and efficient type of hypervisor, which is why they dominate in data centres and cloud services.

  • Pros:
    • Efficiency: Direct hardware access means near-native performance.
    • Security: Fewer layers reduce the attack surface.
    • Scalability: Designed for running hundreds or thousands of VMs.
  • Cons:
    • Complex setup: Requires specialist knowledge to install and configure.
    • Dedicated hardware: Cannot usually run alongside a standard desktop OS.
    • Cost: Enterprise solutions (e.g., VMware ESXi) can be expensive.

Extra detail: Cloud providers (AWS EC2, Microsoft Azure, Google Cloud) rely on Type 1 hypervisors to provide their customers with secure, isolated virtual machines.

Type 2 (Hosted)

These run on top of an existing operating system. The hypervisor behaves like an application, making it more user-friendly but less efficient. They are ideal for students, hobbyists, and developers.

  • Pros:
    • User-friendly: Easy to install and run like normal software.
    • Accessible: Can run on standard laptops or desktops.
    • Great for testing: Allows users to experiment with multiple operating systems.
  • Cons:
    • Performance loss: The host OS consumes resources, slowing down VMs.
    • Less secure: If the host OS is compromised, so are the VMs.
    • Not enterprise-ready: Unsuitable for large-scale deployment.

Extra detail: Developers often use Type 2 hypervisors (like VirtualBox or VMware Workstation) to simulate multi-OS environments, e.g., testing how a website looks in both Windows and Linux without switching machines.

In summary, virtual machines provide flexibility and isolation, while hypervisors are the backbone that makes them possible. Type 1 is used in enterprise and cloud systems, while Type 2 is suited to personal or small-scale use.

7.4.2 - Understand the key features of virtual environments

Virtual environments bring powerful features that improve flexibility, security, and efficiency. These features explain why virtualisation is at the core of modern IT systems, from cloud computing to application development.

Increased Security

Virtual environments enhance security by isolating applications and systems. If one virtual machine is compromised, the threat does not usually spread to others or the host system.

  • Pros:
    • Containment: Malware or crashes stay confined to a single VM.
    • Safe testing: Developers can test unknown or risky code in a sandbox.
    • Segregation: Sensitive applications can run separately from general workloads.
  • Cons:
    • False sense of safety: If not configured properly, attacks can still escape the VM.
    • Overhead: Security features may reduce performance.
    • Shared vulnerabilities: If the hypervisor itself is attacked, all VMs are at risk.

Managed Execution

Virtual environments allow controlled execution of programs. The hypervisor or virtual machine manager monitors processes, allocating resources fairly.

  • Pros:
    • Efficiency: CPU, memory, and I/O usage are carefully balanced.
    • Stability: A misbehaving process in one VM won’t bring down the whole system.
    • Monitoring: Administrators can easily track and limit resource usage.
  • Cons:
    • Resource overhead: Monitoring and scheduling add extra processing load.
    • Complexity: Requires skilled management to avoid bottlenecks.

Sharing

Virtual environments allow multiple VMs to share hardware resources such as CPUs, memory, and storage. This makes better use of a single physical machine.

  • Pros:
    • Cost efficiency: Reduces the need for multiple physical servers.
    • Collaboration: Different users or teams can share resources safely.
    • Flexibility: Resources can be reallocated as workloads change.
  • Cons:
    • Contention: Performance may drop if too many VMs compete for the same resource.
    • Management overhead: Requires policies to prevent resource hogging.

Aggregation

Aggregation combines resources from multiple machines into one virtualised pool. This makes it easier to scale services and balance workloads.

  • Pros:
    • Scalability: Resources can be added without major disruption.
    • High availability: Workloads can be shifted if one machine fails.
    • Unified management: Administrators view resources as a single pool.
  • Cons:
    • Complex setup: Requires advanced virtualisation tools.
    • Single point of failure: If aggregation software fails, many systems may be affected.

Emulation

Emulation allows one system to mimic another. For example, running Windows-only software on a Mac using a virtual machine.

  • Pros:
    • Legacy support: Old applications can run on modern systems.
    • Cross-platform compatibility: Software built for one OS can run on another.
    • Testing flexibility: Developers can test across multiple platforms without extra hardware.
  • Cons:
    • Performance drop: Emulation is slower than native execution.
    • Limited hardware access: Some specialised hardware may not be emulated well.

Isolation

Isolation keeps each VM separate from others and from the host. This prevents interference and improves stability.

  • Pros:
    • Fault tolerance: Crashes in one VM don’t affect others.
    • Security: Sensitive workloads can be isolated from general ones.
    • Experimentation: Users can test safely without harming the host system.
  • Cons:
    • Resource duplication: Each VM needs its own OS and drivers, increasing storage use.
    • Overhead: More layers = more resource consumption.

Portability

Virtual machines are portable — they can be copied or moved between systems easily. A VM is often stored as a single file, making migration and backup straightforward.

  • Pros:
    • Easy migration: VMs can be moved between machines or data centres.
    • Disaster recovery: Quick restoration from a VM image after failure.
    • Consistency: Identical environments can be deployed across many systems.
  • Cons:
    • File size: VM images can be very large, requiring significant storage.
    • Compatibility issues: A VM may not run properly on all hardware.

In summary: These features make virtual environments more secure, flexible, and adaptable — but they can come with trade-offs such as performance overhead and complexity.

7.4.3 - Understand the benefits of the use of virtual environments

Virtual environments provide many advantages for organisations, schools, and individuals. They improve efficiency, reduce costs, and create safer and more flexible computing systems.

Cost Effectiveness for Large Environments

Instead of buying and maintaining multiple physical servers, organisations can run many virtual machines on one powerful machine.

  • Pros:
    • Reduced hardware costs: Fewer physical servers are required.
    • Lower running costs: Less energy is needed for power and cooling.
    • Licensing savings: Many virtualisation solutions are open-source or cheaper than hardware expansion.
  • Cons:
    • Initial investment: High-performance servers and hypervisors can be expensive upfront.
    • Management skills: Requires trained staff to configure and monitor efficiently.

Easy Management

Virtual environments make it simpler to manage resources, apply updates, and monitor system health.

  • Pros:
    • Centralised control: Admins can manage multiple VMs from one console.
    • Automation: Tasks such as updates, scaling, and monitoring can be automated.
    • Rapid deployment: New servers or environments can be created in minutes.

Resilience

Virtual systems can be made highly resilient by running multiple copies of critical services or quickly switching workloads if one VM fails.

  • Pros:
    • High availability: Failover mechanisms ensure services keep running.
    • Load balancing: Traffic can be spread across multiple VMs for stability.
    • Self-healing systems: Damaged VMs can be restarted or replaced quickly.

Potentially Lower Carbon Footprint

Fewer physical machines means less energy consumption, which is better for the environment.

  • Pros:
    • Reduced electricity usage: Less hardware running 24/7.
    • Smaller cooling needs: Data centres can reduce air conditioning requirements.
    • Sustainable IT: Organisations can demonstrate environmentally friendly practices.

Improved Disaster Recovery Options

Virtualisation allows easy backup and recovery of systems since VMs can be stored as files.

  • Pros:
    • Quick restoration: Damaged systems can be restored in minutes from backups.
    • Off-site storage: VM images can be stored in the cloud for extra safety.
    • Replication: Live VMs can be mirrored in real-time for rapid failover.

Better Testing Environments

Virtualisation makes it easy to create safe test environments for software, operating systems, and security research.

  • Pros:
    • Sandboxing: Risky software can be tested without harming the host.
    • Snapshots: Systems can be rolled back to a clean state instantly.
    • Multi-platform testing: Developers can run different OSs on the same machine.

Provision of Education and Training Options

Virtual environments are valuable in teaching and training because students can access isolated, preconfigured systems without needing physical hardware.

  • Pros:
    • Hands-on learning: Students can practice on realistic systems safely.
    • Accessibility: Training environments can be deployed remotely to learners anywhere.
    • Cost savings: Schools and training providers avoid the cost of multiple physical PCs.

In summary: Virtual environments improve efficiency, reduce costs, and enable safer testing and training, while also contributing to sustainability and disaster resilience.

7.4.4 - Understand the drawbacks of the use of virtual environments

What is a Virtual Environment?

Before we discuss the drawbacks, a quick recap:

A virtual environment is a software-based simulation of a computer system. Examples include virtual machines (VMs) like VirtualBox, VMware, or container-based environments like Docker. They let you run an operating system or application in isolation from your physical hardware.

Extra Hardware Load

What it means: Running a virtual environment uses additional system resources (CPU, RAM, and storage) on top of what your host system already needs.

Why it matters: Even if your computer is powerful, running a VM or container consumes extra resources. If your host system is not very strong, this can slow down both the virtual environment and your main system.

Example: Imagine you have a laptop with 8 GB RAM. Running Windows 10 in a VM might require 4 GB, leaving only 4 GB for your host system. Your laptop could feel sluggish.

Slower Execution Time

What it means: Programs running inside a virtual environment usually run slower than on the host system.

Why it happens: Virtual environments need to translate commands from the guest OS to the host OS, a process called virtualization overhead.

Example: A game or heavy data processing task might take 20-30% longer in a VM than running directly on your computer.

Potential for False Representation of Performance

What it means: The performance you see inside a virtual environment may not match real-world performance on actual hardware.

Why it happens: Because the virtual environment doesn't have direct access to all the host's hardware features, some programs may behave differently or run faster/slower than they would on a real machine.

Example: Testing a server application in a VM might show it handling 500 users easily, but when deployed on a real server, it may crash under the same load due to hardware differences.
