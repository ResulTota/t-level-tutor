Subtopic 7.2: Software

7.2.1 - Understand the features and use of operating systems

An Operating System (OS) is system software that manages computer hardware, software resources, and provides services for computer programs. There are different types of operating systems designed for different tasks and environments.

Batch Operating System

Features:

  • Non-interactive applications: Jobs are collected, grouped, and executed without user interaction during processing.
  • High volume: Handles large amounts of data and repetitive tasks efficiently.
  • Scheduling: Jobs are queued and executed in order, often prioritised by the system.

Uses:

  • Payroll systems.
  • Banking transaction processing.
  • Data analysis on large datasets.

Multitasking Operating System

Features:

  • Concurrent execution of multiple tasks: Runs more than one program at a time.
  • Time-slicing: Processor time is divided between tasks, giving the appearance of simultaneous execution.
  • Interrupts: Mechanism that allows the OS to pause one task and switch to another when needed.

Uses:

  • Desktop and laptop operating systems (Windows, macOS, Linux).
  • Running multiple applications like web browsers, word processors, and media players at the same time.

Real-Time Operating System (RTOS)

Features:

  • Monitoring and control applications: Provides predictable response times.
  • Transaction processing: Handles critical operations where timing is crucial.

Uses:

  • Embedded systems (e.g., car engine management, robotics).
  • Medical devices requiring instant feedback.
  • Industrial automation and control systems.

Network Operating System

Features:

  • Resource sharing: Manages file and printer sharing across a network.
  • User management: Provides authentication, permissions, and account control.
  • Communication: Facilitates messaging, data transfer, and remote access between devices.

Uses:

  • Client-server environments (Windows Server, Linux server distros).
  • Corporate networks requiring centralised control and security.

Mobile Operating System

Features:

  • Smartphones and tablets: Designed specifically for mobile devices.
  • Lower processing requirements: Optimised to run efficiently on less powerful hardware.
  • Increased battery life: Power management is a core feature.

Uses:

  • Mobile devices such as Android phones and iPhones (iOS).
  • Supports app ecosystems, touch input, and wireless connectivity.

Summary

Different types of operating systems are tailored for specific tasks: batch OS for repetitive processing, multitasking OS for general computing, real-time OS for time-critical applications, network OS for resource sharing, and mobile OS for portable devices.

7.2.2 - Understand the features and use of common utilities

Utilities are system tools that support the operation, maintenance, and management of a computer system. They perform specific tasks that enhance performance, security, and reliability.

File Management

Features:

  • Provides the ability to create, edit, move, copy, and delete files.
  • Organises files into folders/directories for easier navigation.
  • Supports file permissions and access control.

Uses:

  • Managing personal documents, images, and videos.
  • System administrators organising large datasets.
  • Accessing files stored locally or on network drives.

Defragmenters

Features:

  • Reorganises fragmented data on magnetic hard drives.
  • Improves file access speed by storing data in contiguous blocks.
  • Reduces unnecessary movement of the disk head.

Uses:

  • Improving system performance on HDDs.
  • Reducing wear and tear on mechanical drives.

Note: Not required for SSDs, as they use flash memory.

File Compression

Features:

  • Reduces file size using compression algorithms (e.g., ZIP, RAR).
  • Can be lossless (no data lost) or lossy (reduced quality for smaller size).

Uses:

  • Saving storage space.
  • Speeding up file transfers and downloads.
  • Archiving large collections of data.

Package Managers

Features:

  • Automates installation, updating, and removal of software.
  • Resolves dependencies between software packages.
  • Ensures software is securely verified and up-to-date.

Uses:

  • Linux distributions (e.g., apt, yum, pacman).
  • Programming environments (e.g., npm for Node.js, pip for Python).
  • Managing software libraries and frameworks efficiently.

Protection Software

Features:

  • Antivirus to detect and remove malicious software.
  • Firewalls to monitor and block unauthorised network access.
  • Encryption tools to secure sensitive data.

Uses:

  • Protecting systems against malware, spyware, and ransomware.
  • Ensuring data security for personal and business use.
  • Preventing unauthorised access to networks.

Backup Software

Features:

  • Creates copies of files or entire systems.
  • Supports incremental, differential, and full backups.
  • Often includes automated scheduling of backups.

Uses:

  • Protecting against data loss due to hardware failure or malware.
  • Recovering previous versions of files.
  • Ensuring business continuity and disaster recovery.

Understand the features and use of common code development tools

Code development tools support programmers in writing, testing, and maintaining software. They improve productivity and reduce errors during the software development lifecycle.

Integrated Development Environments (IDEs)

Features:

  • Code editing tools - syntax highlighting, auto-completion, code navigation.
  • Debugging tools - breakpoints, step-through execution, variable inspection.
  • Screen design tools - drag-and-drop interfaces for UI/UX development.

Uses:

  • Providing an all-in-one environment for coding, testing, and debugging.
  • Reducing errors with real-time suggestions and error detection.
  • Speeding up development of desktop, mobile, and web applications.

Compilers

Features:

  • Translate high-level code into machine code.
  • Provide error reports before execution.
  • Optimise code for performance.

Uses:

  • Creating standalone executable programs.
  • Ensuring programs run efficiently on target hardware.

Interpreters

Features:

  • Execute code line by line instead of compiling into a full program first.
  • Provide immediate feedback during development.
  • Easier debugging with interactive testing.

Uses:

  • Running scripts quickly without compiling.
  • Useful for scripting languages (e.g., Python, JavaScript).
  • Rapid prototyping and testing ideas.

7.2.4 - Understand the features and use of common application software

Application software allows users to perform specific tasks, such as creating documents, managing data, or communicating online.

Word Processors

Features:

  • Create, edit, and format text documents.
  • Support for images, tables, and templates.
  • Spelling, grammar, and collaboration tools.

Uses:

  • Writing letters, essays, and reports.
  • Collaborating on shared documents.

Spreadsheets

Features:

  • Organise data in rows and columns.
  • Support for formulas, charts, and pivot tables.
  • Automation with macros.

Uses:

  • Financial planning and budgeting.
  • Data analysis and visualisation.

Databases

Features:

  • Structured storage of large volumes of data.
  • Support for queries and reports.
  • Enforces data integrity and security.

Uses:

  • Managing customer records and business data.
  • Supporting websites and applications with back-end data.

Email

Features:

  • Send and receive digital messages.
  • Supports attachments and cloud integration.
  • Filters, spam protection, and contact management.

Uses:

  • Business communication and collaboration.
  • Personal correspondence and notifications.

Project Management Software

Features:

  • Task assignment, scheduling, and progress tracking.
  • Supports collaboration and resource allocation.
  • Visual tools such as Gantt charts and Kanban boards.

Uses:

  • Managing software development projects.
  • Coordinating teams and deadlines.
