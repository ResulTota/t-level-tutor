Subtopic 7.3: Networks

7.3.1 - Benefits and Drawbacks of Connecting Devices to Form Networks

Connecting devices into networks enables communication, resource sharing and centralised management â€” but it also introduces costs, complexity and security risks. Below we outline the main benefits and drawbacks so you can evaluate when and how to use networks effectively.

Benefits

  • Resource sharing â€” share printers, files, storage and internet access across many devices, reducing cost and duplication.
  • Improved communication & collaboration â€” email, chat, video conferencing and collaborative tools let users work together in real time.
  • Centralised management â€” administrators can deploy updates, manage backups and enforce policies from one place.
  • Scalability â€” networks make it easier to add new devices and services without redesigning the whole system.
  • Remote access â€” users can access resources from different locations (often via secure methods like VPNs).

Drawbacks

  • Security risks â€” connected systems are exposed to malware, unauthorised access and data breaches unless properly protected.
  • Costs â€” initial hardware, cabling, licensing and configuration costs can be significant.
  • Maintenance & administration â€” networks require ongoing monitoring, troubleshooting and skilled IT staff.
  • Single points of failure â€” if key network equipment or links fail, many users can lose access until repaired.
  • Performance constraints â€” network congestion, latency or poor configuration can reduce application responsiveness.

7.3.2 - Features of Different Types of Networks

This section covers the main features used to describe networks, then explains specific network types: PAN, LAN, MAN, WAN and VPN. For each type we describe coverage, typical number of users, common connection media and typical uses.

Key characteristics to consider

  • Number of users â€” ranges from a single person (PAN) to millions (global WAN/Internet).
  • Connection media â€” wired (Ethernet, fibre) or wireless (Wi-Fi, Bluetooth, cellular, satellite).
  • Coverage area â€” physical distance the network spans (room, building, city, country, globe).
  • Topology & architecture â€” how devices are logically and physically connected (star, mesh, hierarchical, client-server).
  • Security & access control â€” how access is restricted and data protected (encryption, firewalls, authentication).

Personal Area Network (PAN)

  • Coverage: a few metres (typically around one person).
  • Number of users: usually a single user with several personal devices.
  • Connection media: Bluetooth, NFC, sometimes USB or wireless personal hotspots.
  • Typical uses: connecting phone to headphones, smartwatch to phone, or syncing a laptop to a phone's hotspot.
  • Notes: PANs are simple and very local; security is usually lightweight but still important (pairing, PINs).

Local Area Network (LAN)

  • Coverage: single building or site (home, office, school).
  • Number of users: tens to hundreds.
  • Connection media: Ethernet (copper/fibre), Wi-Fi for wireless access.
  • Typical uses: file sharing, printing services, local servers, corporate or campus networks.
  • Notes: LANs usually provide high throughput and low latency; they are commonly managed by a central IT team in larger organisations.

Metropolitan Area Network (MAN)

  • Coverage: city-wide or large campus areas.
  • Number of users: hundreds to thousands.
  • Connection media: high-capacity fibre, microwave links or leased lines.
  • Typical uses: connecting multiple buildings of a city university, linking corporate sites across a city, municipal services.
  • Notes: MANs bridge several LANs â€” they are larger than LANs but smaller than WANs and often operated by ISPs or local authorities.

Wide Area Network (WAN)

  • Coverage: national or international (country to globe).
  • Number of users: thousands to millions.
  • Connection media: fibre-optic backbones, submarine cables, satellite links, leased circuits spanning long distances.
  • Typical uses: the Internet, multinational company networks, cloud provider backbones.
  • Notes: WANs usually involve multiple service providers and focus on long-distance connectivity rather than local high-speed access.

Virtual Private Network (VPN)

  • Coverage: virtual â€” can span any geographic distance using underlying public networks.
  • Number of users: variable â€” small teams to large organisations with many remote users.
  • Connection media: encrypted tunnels over Internet/WAN links (IPsec, SSL/TLS-based VPNs, WireGuard, etc.).
  • Typical uses: secure remote access to corporate resources, connecting branch offices securely over public infrastructure.
  • Notes: VPNs provide confidentiality and integrity for data in transit, but depend on strong authentication and up-to-date cryptography to be secure.

Summary

Networks let devices share resources and communicate at different scales â€” from a single user's PAN to the global WAN/Internet. Choosing the right network type and technologies depends on required coverage, expected number of users, performance needs and security constraints. Understanding these trade-offs helps you design and use networks that are fit-for-purpose.

7.3.3 - Understand the features, characteristics, benefits and drawbacks of connectivity methods

Connectivity methods describe how devices physically or wirelessly connect to a network. Each method has unique features, benefits and drawbacks that influence performance, reliability and cost.

Wired Connectivity

Copper / Ethernet
  • Features: Uses twisted-pair copper cables (Cat5e, Cat6, Cat7). Common in LANs.
  • Pros:
    • Low cost compared to fibre-optic.
    • Widely available and easy to install.
    • Good speeds up to 10 Gbps.
  • Cons:
    • Limited distance (about 100m before repeaters/switches are needed).
    • Susceptible to interference from electromagnetic signals.
    • Lower bandwidth compared to fibre-optic.
Fibre-Optic
  • Features: Uses glass or plastic fibres to transmit data as light signals.
  • Pros:
    • Very high speeds (Gbpsâ€“Tbps).
    • Long-distance transmission with minimal signal loss.
    • Immune to interference from electromagnetic signals.
  • Cons:
    • Expensive compared to copper cabling.
    • Requires specialist installation and maintenance.
    • Fragile cables that are harder to repair.

Wireless Connectivity

Wireless Access Points (WAPs)
  • Features: Provide wireless connectivity to devices using Wi-Fi standards (IEEE 802.11).
  • Pros:
    • Mobility for connected devices.
    • Flexible with no need for cables at endpoints.
    • Scalable â€” new devices can be added easily.
  • Cons:
    • Security risks due to interception and unauthorised access.
    • Signal interference from walls and other devices.
    • Lower bandwidth and less consistent speed compared to wired.

7.3.4 - Understand the features, benefits and drawbacks of the common network topologies

Topology refers to how devices are arranged and connected in a network. Networks can be described by their physical topology (cable/device layout) or their logical topology (how data actually flows between devices).

Star Topology

  • Features: All devices connect to a central switch or hub.
  • Pros:
    • Easy to manage and set up.
    • Fault isolation â€” one device failing doesnâ€™t affect others.
    • Scalable â€” devices can be added or removed easily.
  • Cons:
    • Single point of failure at the hub/switch.
    • More cabling required than simpler topologies.
    • Higher cost due to extra equipment.

Mesh Topology

  • Features: Every device connects to multiple other devices, forming a web.
  • Pros:
    • High reliability â€” multiple routes for data.
    • Redundant connections improve fault tolerance.
    • Scalable for critical systems requiring uptime.
  • Cons:
    • Complex setup requiring advanced configuration.
    • Expensive due to more cabling and hardware.
    • Maintenance challenges as the network grows.

Tree Topology

  • Features: Hierarchical structure combining multiple star networks in a branching arrangement.
  • Pros:
    • Scalable for large networks.
    • Structured hierarchy simplifies management.
    • Combination of star and bus advantages.
  • Cons:
    • Backbone failure can affect entire sections.
    • Complex maintenance in large networks.
    • High cabling cost compared to simpler designs.

Logical vs Physical Topologies

  • Physical topology: The actual physical layout of cables, switches, routers and devices.
  • Logical topology: How data flows through the network, regardless of physical design.
  • Example: A physically star-shaped network (all devices connected to a switch) may have a logical bus topology if data is broadcast to all devices.

7.3.5 - Network Models

Different organizations use different types of networks depending on cost, control, and usage needs.

1. Client-Server Model

Features:

  • One central server provides resources (files, applications, storage) to multiple clients (computers, devices).
  • Clients request services, the server responds.
  • Examples: Email servers, database servers, file servers.

Benefits:

  • Centralized control of data, security, and backups.
  • Easy management for large networks.
  • Scalable â€” can add more clients easily.

Drawbacks:

  • Expensive to set up (servers are costly).
  • Single point of failure â€” server failure = loss of service for all clients.
  • Requires skilled administrators.

2. Thin Client Model

Features:

  • Clients are "lightweight" devices that depend on a server for most processing.
  • The server does the heavy work (applications, processing, storage).
  • Clients are basically input/output devices.

Benefits:

  • Low-cost client machines (less powerful hardware needed).
  • Centralized updates and maintenance.
  • Secure, since data is stored on the server not the client.

Drawbacks:

  • Heavy server load - server must be powerful.
  • Requires a fast, reliable network connection.
  • Clients depend on the server - poor performance if disconnected.

3. Peer-to-Peer (P2P) Model

Features:

  • All computers (peers) are equal - they act as both client and server.
  • Each device can share files and resources directly with others.
  • No central server.

Benefits:

  • Inexpensive â€” no dedicated server required.
  • Easy to set up for small networks.
  • Good for file sharing in small groups.

Drawbacks:

  • No central control â€” harder to manage security and backups.
  • Performance issues as the network grows.
  • Higher risk of malware spread since all devices are interconnected.

Quick Comparison Table

Model Features Benefits Drawbacks
Client-Server Central server provides resources Centralized control, scalable, secure Expensive, single point of failure, admin needed
Thin Client Clients rely on server for processing Cheaper clients, easy updates, secure Heavy server load, needs strong network, server dependency
Peer-to-Peer Devices equal, share resources directly Cheap, easy setup, good for small groups No central control, less secure, not scalable

7.3.6 - Common Network Components

Every network, from a small home setup to the global Internet, relies on key components. Each has a specific role in enabling communication and data transfer.

1. Server

Role: A powerful computer that provides services, resources, or data to other computers (clients) on the network.

Examples:

  • File server - stores and manages documents
  • Mail server - handles email sending and receiving
  • Web server - hosts websites and delivers web pages

2. Client

Role: A device (PC, laptop, phone, tablet) that requests services or resources from a server.

Examples:

  • A student's laptop accessing files from a school's file server.
  • A smartphone requesting a webpage from a web server.

3. Router

Role:

  • Directs data between different networks.
  • Connects a local network (LAN) to the Internet (WAN).
  • Assigns IP addresses to devices.

Example: Your home broadband router connects all your devices to the Internet.

4. Network Switch

Role:

  • Connects multiple devices within a local network (LAN).
  • Forwards data only to the specific device it is intended for (efficient compared to a hub).

Example: In an office, a switch connects computers, printers, and servers together.

5. Internet Connection / Internet Backbone

Internet connection:

  • Provides access from a local network to the global Internet.
  • Supplied by an Internet Service Provider (ISP).

Internet backbone:

  • The core infrastructure of the Internet.
  • Made up of high-speed fiber-optic cables, routers, and data centers operated by major telecom companies.
  • Carries the majority of Internet traffic across countries and continents.

7.3.7 - Understand the seven-layer OSI (Open Systems Interconnection) model

The OSI model is a conceptual framework that standardises how different networking devices and systems communicate. It divides communication into seven layers, each with specific functions and related protocols.

Layer Function Examples of Protocols
Application (Layer 7) Provides services and interfaces for end-user applications (e.g. web browsers, email clients). HTTP, HTTPS, FTP, SMTP, POP3
Presentation (Layer 6) Translates data formats, handles encryption, compression, and character encoding. SSL/TLS, JPEG, MPEG
Session (Layer 5) Manages and maintains sessions (start, manage, end) between communicating devices. NetBIOS, RPC
Transport (Layer 4) Ensures reliable data delivery, error checking, segmentation, and flow control. TCP, UDP
Network (Layer 3) Handles logical addressing and routing so data can travel across networks. IP, ICMP, IPsec
Data Link (Layer 2) Provides physical addressing (MAC addresses), error detection, and frame delivery within a LAN. Ethernet, Wi-Fi, PPP
Physical (Layer 1) Transmits raw bits over the physical medium (cables, radio waves, fibre). Ethernet cables, Fibre optics, Wi-Fi radio signals

Mnemonic: A common way to remember the OSI layers from top (7) to bottom (1) is: All People Seem To Need Data Processing.

7.3.8 - Understand the four-layer TCP/IP (Transmission Control Protocol/Internet Protocol) model

The TCP/IP model is a simplified framework for computer networking, used in real-world internet communications. It consists of four layers, each responsible for specific aspects of data transmission.

Layer Function Examples of Protocols
Application Layer Provides network services directly to user applications, including data formatting, encryption, and communication interfaces. HTTP, HTTPS, FTP, SMTP, DNS
Transport Layer Ensures reliable delivery of data, segmentation, error detection, and flow control between devices. TCP, UDP
Internet Layer Handles logical addressing and routing so that data packets can travel across interconnected networks. IP, ICMP, ARP
Network Access / Link Layer Responsible for the physical transmission of data over the network hardware and manages local addressing (MAC addresses). Ethernet, Wi-Fi, PPP

Note: The TCP/IP model is widely used in practice and maps roughly to the OSI model: OSI layers 7â€“5 correspond to TCP/IP Application, OSI 4 = TCP/IP Transport, OSI 3 = TCP/IP Internet, and OSI 2â€“1 = TCP/IP Network Access.

7.3.9 - Understand the role of data packets in transmitting over a network

Data packets are small units of data transmitted across a network. Breaking information into packets allows efficient, reliable communication and makes it easier to manage errors and routing.

Contents and Structure of a Data Packet

  • Header: Contains control information such as source and destination addresses, protocol type, and sequencing information.
  • Payload / Data: The actual content being transmitted, such as part of a file, message, or request.
  • Trailer: Includes error-checking information such as a checksum or cyclic redundancy check (CRC).

Role of the Components of a Data Packet

  • Source and Destination Addresses: Identify where the packet comes from and where it should go.
  • Sequence Number: Allows the receiving device to reassemble packets in the correct order.
  • Error-Checking Data: Detects if the packet has been corrupted during transmission.

Packet Switching

Packet switching is a method where data is divided into packets and sent independently across the network. Each packet may take a different route to the destination.

  • Causes of Packet Loss: Network congestion, faulty hardware, corrupted packets, or transmission errors can cause some packets to be lost.

Error Handling

Networks use error detection and correction techniques to ensure data integrity.

  • Cyclic Redundancy Check (CRC): A mathematical calculation added to the packet trailer. The receiver recalculates the CRC and compares it with the sent value to detect errors.

Understanding packets, switching, and error handling is essential for designing reliable networks and troubleshooting communication issues.

7.3.10 - Understand the role of common network protocols

Network protocols define rules and conventions for communication between devices over a network. Different protocols serve specific purposes, such as web browsing, email delivery, routing, or file transfer.

Web Protocols

  • HTTP (HyperText Transfer Protocol): The standard protocol used for transferring web pages over the internet. Works without encryption.
  • HTTPS (HTTP Secure): An extension of HTTP that adds encryption (TLS/SSL) to ensure secure communication between browsers and web servers.

Mail Protocols

  • SMTP (Simple Mail Transfer Protocol): Used to send emails from a client to a mail server or between mail servers.
  • POP (Post Office Protocol): Used by email clients to retrieve messages from a server, typically downloading and removing them from the server.
  • IMAP (Internet Message Access Protocol): Retrieves and manages email on the server, allowing multiple devices to access the same mailbox.

Routing Protocols

  • RIP (Routing Information Protocol): A simple distance-vector routing protocol used to determine the best path based on hop count.
  • OSPF (Open Shortest Path First): A more advanced link-state routing protocol that uses path cost and network topology to find the most efficient route.

Application Protocols

  • FTP (File Transfer Protocol): Transfers files between client and server over a network.
  • SFTP (Secure File Transfer Protocol): A secure version of FTP that uses encryption to protect data during transfer.
  • DHCP (Dynamic Host Configuration Protocol): Automatically assigns IP addresses and network configuration to devices on a network.
  • DNS (Domain Name System): Translates human-readable domain names (like www.example.com) into IP addresses that devices can use for routing.

Understanding these protocols is essential for managing communication, security, and data flow across networks.

7.3.11 - Understand the concepts of bandwidth and latency, and their effect on the performance of networks and connected systems

Network performance depends on several factors, with bandwidth and latency being two of the most important. Both directly impact the speed and responsiveness of data transmission between devices.

Bandwidth

  • Definition: The maximum amount of data that can be transmitted over a network in a given time, usually measured in bits per second (bps), kilobits (Kbps), megabits (Mbps), or gigabits (Gbps).
  • Effect on performance: Higher bandwidth allows more data to be sent simultaneously, improving download/upload speeds and overall network throughput.
  • Example: Streaming a high-definition video requires higher bandwidth than sending a simple text message.

Latency

  • Definition: The time it takes for a data packet to travel from the source to the destination, measured in milliseconds (ms).
  • Effect on performance: High latency causes delays in communication, which can affect real-time applications like online gaming, video calls, and VoIP.
  • Factors affecting latency: Distance between devices, network congestion, routing inefficiencies, and processing delays in hardware/software.

Interaction Between Bandwidth and Latency

  • Even with high bandwidth, high latency can make applications feel slow.
  • Optimising both bandwidth and latency is important for a smooth user experience.
  • Network engineers often balance bandwidth allocation and reduce latency to improve performance for critical applications.

Understanding bandwidth and latency helps in designing, troubleshooting, and optimising networks and connected systems to meet performance requirements.
