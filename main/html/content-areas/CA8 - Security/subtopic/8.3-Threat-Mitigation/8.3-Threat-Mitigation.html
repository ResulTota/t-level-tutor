Subtopic 8.3: Threat Mitigation

8.3.1 - Understand the purposes, processes, benefits and drawbacks of common threat mitigation techniques.

Threat mitigation uses strategies to lower the chance and impact of security threats to information systems. It involves identifying risks, applying controls to prevent or limit attacks, and monitoring systems to quickly detect and respond to incidents.

  • Security Settings:
    • Hardware: Physical controls (e.g., firewalls, biometric access) to restrict unauthorized access.
    • Software: Configuration of operating systems and applications to enforce security policies.
    Benefits: Reduces attack surface and prevents unauthorized changes.
    Drawbacks: Misconfiguration can create vulnerabilities.

  • Anti-malware Software:
    • Function: Detects and removes malicious software.
    • Actions: Scans files, quarantines threats, and provides real-time protection.
    Benefits: Protects systems from viruses, trojans, and other malware.
    Drawbacks: May impact system performance and require regular updates.

  • Intrusion Detection:
    Monitors network or system activities for suspicious behavior. Benefits: Early warning of attacks.
    Drawbacks: False positives can cause unnecessary alerts.

  • Encryption:
    • Hashing: Secures data integrity by generating unique values for data.
    • Symmetric: Uses the same key for encryption and decryption.
    • Asymmetric: Uses public and private keys for secure communication.
    Benefits: Protects data confidentiality and integrity.
    Drawbacks: Key management can be complex.

  • User Access Policies:
    Defines who can access what resources and at what level.
    Benefits: Limits exposure of sensitive data.
    Drawbacks: Overly restrictive policies can hinder productivity.

  • Staff Vetting:
    Background checks to ensure trustworthy personnel.
    Benefits: Reduces insider threats.
    Drawbacks: Can be time-consuming and costly.

  • Staff Training:
    Educates employees on security best practices.
    Benefits: Reduces risk of human error.
    Drawbacks: Requires ongoing investment.

  • Software-based Access Control:
    Uses software to manage user permissions and authentication.
    Benefits: Flexible and scalable.
    Drawbacks: Vulnerable to software exploits.

  • Device Hardening:
    Secures devices by disabling unnecessary features and services.
    Benefits: Reduces vulnerabilities.
    Drawbacks: May limit device functionality.

  • Backups:
    • Type: Full, incremental, differential backups.
    • Safe Storage: Offsite or encrypted storage to protect backup data.
    Benefits: Enables recovery from data loss.
    Drawbacks: Requires storage space and regular maintenance.

  • Software Updates:
    Regularly applying patches to fix vulnerabilities.
    Benefits: Protects against known threats.
    Drawbacks: May cause compatibility issues.

  • Firmware/Driver Updates:
    Keeps hardware components secure and functional.
    Benefits: Fixes hardware vulnerabilities.
    Drawbacks: Risk of device instability.

  • Air Gaps:
    Physically isolates systems from unsecured networks.
    Benefits: Prevents remote attacks.
    Drawbacks: Limits usability and data transfer.

  • Certification of APIs:
    Ensures APIs meet security standards before use.
    Benefits: Reduces risk of insecure integrations.
    Drawbacks: Can delay deployment.

  • VPNs (Virtual Private Networks):
    Encrypts network traffic for secure remote access.
    Benefits: Protects data in transit.
    Drawbacks: May slow down connections.

  • Multi-factor Authentication:
    Requires multiple forms of verification for access.
    Benefits: Stronger protection against unauthorized access.
    Drawbacks: Can be inconvenient for users.

  • Password Managers:
    Securely stores and manages user passwords.
    Benefits: Encourages strong, unique passwords.
    Drawbacks: Single point of failure if compromised.

  • Port Scanning:
    Identifies open ports and potential vulnerabilities.
    Benefits: Helps detect weaknesses.
    Drawbacks: Can be used by attackers for reconnaissance.

  • Penetration Testing:
    • Ethical Hacking: Authorized testing to find vulnerabilities.
    • Unethical Hacking: Unauthorized attacks to exploit weaknesses.
    Benefits: Identifies and helps fix security gaps.
    Drawbacks: Can be costly and disruptive.

Processes and procedures that ensure internet security & reasons why they are used:

  • Firewall Configuration:
    • Rules for Traffic (Inbound and Outbound): Controls which data packets are allowed to enter or leave the network.
    • Traffic Type Rules: Filters traffic based on protocols (e.g., HTTP, FTP, SMTP).
    • Application Rules: Restricts or allows network access for specific applications.
    • IP Address Rules: Allows or blocks traffic from specified IP addresses or ranges.
    Benefits: Prevents unauthorized access and reduces exposure to threats.
    Drawbacks: Incorrect rule configuration can block legitimate traffic or allow malicious activity.

  • Network Segregation:
    • Virtual: Uses VLANs to separate network segments for security and management.
    • Physical: Isolates networks using separate hardware and cabling.
    • Offline Network: Keeps sensitive systems disconnected from external networks.
    Benefits: Limits the spread of attacks and protects critical assets.
    Drawbacks: Can increase complexity and cost.

  • Network Monitoring:
    Continuously observes network activity to detect suspicious behavior or security incidents.
    Benefits: Enables rapid response to threats and helps identify vulnerabilities.
    Drawbacks: May generate large amounts of data and require skilled personnel to interpret results.

  • Port Scanning:
    Examines network devices to identify open ports and potential vulnerabilities.
    Benefits: Helps administrators find and secure weak points.
    Drawbacks: Can be used by attackers for reconnaissance; may trigger security alerts.
