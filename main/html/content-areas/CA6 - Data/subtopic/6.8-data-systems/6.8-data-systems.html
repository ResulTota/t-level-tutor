6.8 - Data Systems


Subtopic 6.8: Data Systems

6.8.1 Know the definition of data wrangling and understand its purpose and when it is used

    Definition: Data wrangling, or data munging, is the process of cleaning, transforming, and preparing raw data so that it is in a usable format for analysis or reporting.

    Purpose: Ensures data quality, usability, and reliability for analysis, reporting, and decision-making.

    When it is used: Before analysis, when combining data from multiple sources, or preparing data for automation and machine learning.

    Example: Standardizing sales data from multiple stores that may have inconsistent formats such as "$100" vs "100 USD".

6.8.2 Know and understand the purpose of each step of data wrangling

Structure

    Organize data into logical formats like tables or databases to make it easier to work with and analyze.

    Example: Customer data arranged into columns for name, address, email, and phone number.

Clean

    Remove or correct inaccurate, duplicate, or irrelevant data to ensure reliability.

    Example: Correcting invalid dates or removing repeated entries.

Validate

    Check data against rules or expected patterns to confirm correctness and consistency.

    Example: Ensuring phone numbers have the correct number of digits or emails follow standard formats.

Enrich

    Add additional useful information to improve the value of data.

    Example: Adding geographic data to customer addresses for better analysis.

Output

    Export clean and structured data in formats ready for reporting, dashboards, or analysis.

    Example: Preparing a dataset for sales analysis in Excel or for feeding into a machine learning model.

6.8.3 Know and understand the purpose of each core function of a data system

Input

    Capturing data from users, sensors, or other systems. Accuracy here impacts the entire system.

    Example: Entering sales orders or survey responses.

Search

    Retrieve specific data efficiently from large datasets.

    Example: Searching for all sales in a particular region for last month.

Save

    Store data securely for future use.

    Example: Saving patient records in a hospital database.

Integrate

    Combine multiple datasets from different sources to create a unified view.

    Example: Merging online and in-store sales data.

Organise (Index)

    Structuring and labeling data to allow efficient search and retrieval.

    Example: Indexing a library database by author, title, and publication year.

Output

    Deliver data in a usable format for reports, dashboards, or other systems.

Feedback Loop

    Use outputs to improve inputs and system processes.

    Example: Adjusting input rules after detecting frequent data entry errors.

6.8.4 Know the types of data entry errors and understand how and why they occur

Transcription Errors

    Typing mistakes when transferring data from one source to another, caused by fatigue or distraction.

    Example: Typing “1023” instead of “1032”.

Transposition Errors

    Switching the order of digits or letters, leading to inaccuracies.

    Example: Entering “57” instead of “75”.

6.8.5 Know and understand methods to avoid data entry errors

  • Validation of user input: System checks input against rules to prevent incorrect entries.
  • Verification by double entry: Entering data twice and comparing to detect errors.
  • Drop-down menus: Limits user options to reduce typing errors.
  • Pre-filled data entry boxes: Auto-populated fields reduce manual entry.

6.8.6 Know and understand the factors that impact data entry

    Time needed to create screens: Longer design time can impact overall project schedules.

    Expertise needed to create screens: Skilled personnel ensure effective and user-friendly forms.

    Time needed to enter the data: Longer entry times can lead to fatigue and errors.

6.8.7 Understand the relationship between factors that impact data entry and data quality and make judgements about the suitability of methods to reduce data entry errors in digital software development

    Poorly designed screens or rushed data entry reduce data quality.

    Methods like drop-down menus, pre-filled fields, and validation improve accuracy.

    Double-entry verification is suitable for high-risk or critical data.

6.8.8 Understand the relationship between factors that impact implementation of data entry and make judgements about the suitability of implementing data entry in digital software development

    Implementation factors include development time, complexity, and expertise required.

    Simple data entry forms are easier to implement, while complex validation improves accuracy but requires more resources.

    Good planning balances accuracy, efficiency, and cost for high-quality data entry systems.
