Subtopic 6.4: Data Types

6.4.1 Know the definition of common data types and understand their purpose and when each is used

Integer

    Definition: Whole numbers without decimals (positive or negative).

    Purpose: Used for counting, indexing, or any value that must be whole.

    When used: Storing age, quantity of items, IDs, or loop counters.

    Example: age = 25, items_in_cart = 4

Real (Float)

    Definition: Numbers with decimals (fractional numbers).

    Purpose: Used when precision is required, especially for measurements.

    When used: Storing prices, weight, temperature, or scientific calculations.

    Example: temperature = 36.6, price = 19.99

Character

    Definition: A single letter, number, symbol, or whitespace.

    Purpose: Used for storing single characters.

    When used: Storing a grade, a single initial, or a symbol.

    Example: initial = 'J'

String

    Definition: A sequence of characters.

    Purpose: Used for storing text, words, or alphanumeric information.

    When used: Storing names, addresses, messages, or any textual input.

    Example: name = "Alice", address = "123 Main St"

Boolean

    Definition: Represents two values: True or False.

    Purpose: Used for logical decisions, flags, or conditions.

    When used: Checking if a user is logged in, a light switch is on/off, or if a condition is met.

    Example: is_logged_in = True

Date

    Definition: Represents dates (and sometimes times).

    Purpose: Used for storing calendar information.

    When used: Birthdates, deadlines, transaction timestamps.

    Example: dob = 2000-05-15, order_date = 2025-09-01

Blob (Binary Large Object)

    Definition: Stores large data like images, audio, video, or any binary data.

    Purpose: Used for multimedia storage or any data that isnâ€™t plain text or numbers.

    When used: Saving profile pictures, PDF files, or video clips in databases.

    Example: profile_picture = Blob("image_data")

6.4.2 Understand the interrelationships between structured data, unstructured data and data type

    Structured data: Organized in tables, rows, and columns. Data types are strictly defined. Example: A database table with name (string), age (integer), price (real).

    Unstructured data: Does not follow a predefined structure; can include text, images, videos. Example: Emails, social media posts, videos, and images.

    Interrelationship: Structured data depends on clearly defined data types, while unstructured data may use strings, Blobs, or specialized formats. Proper data types help convert unstructured data into structured formats.

6.4.3 Understand the interrelationships between data type and data transformation

6.4.4 Be able to make judgements about the suitability of using structured data, unstructured data, data types and data transformations in digital software development

    Structured data: Suitable for banking systems, inventory management, employee databases due to predictability and consistency.

    Unstructured data: Suitable for social media platforms, multimedia apps, document storage because of flexibility.

    Data types: Choosing suitable types ensures proper storage and operations:

    • Integer for counts
    • Real for precise measurements
    • String for text
    • Boolean for binary decisions
    • Blob for large multimedia files

    Data transformation: Transform when integrating systems, cleaning data, or performing analytics. Example: Converting string "2025-09-01" to a date type to calculate days until a deadline.
