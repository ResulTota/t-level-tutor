Subtopic 6.6: Structures for storing data

6.6.1 Understand the role of metadata in providing descriptions and contexts for data

Definition and Purpose

    Metadata: Data that describes other data, providing context, meaning, and structure for easier use.

    Descriptive metadata: Identifies and locates data. Example: Title, author, keywords of a research paper.

    Structural metadata: Explains relationships between components. Example: Chapters and sections of a book.

    Administrative metadata: Provides management details. Example: File size, creation date, access permissions, format type.

Importance of Metadata

    Efficient retrieval: Allows systems and users to locate and search data quickly.

    Interoperability: Enables different systems to understand and use the same data correctly.

    Data governance: Tracks ownership, changes, and access rights.

Real-world Example

A digital photograph may include metadata such as camera settings (ISO, aperture, shutter speed), GPS location, date and time, and photographer’s name, making it easier to catalog, search, and edit.

6.6.2 Know the definition of file-based and directory-based structures and understand their purposes and when they are used

File-based Structure

    Data is stored in separate files, each containing related information.

    Each file is independent, with minimal relationships to other files.

Directory-based Structure

    Files are grouped into directories (folders) to organize them hierarchically.

    Subdirectories can further categorize files for better management.

Purpose and Usage

    File-based: Simple, suitable for small datasets or single-user systems. Example: Storing text documents or spreadsheets.

    Directory-based: Scalable and organized for larger datasets. Example:

            Projects/
        ProjectA/
            Budget.xlsx
            Timeline.docx
        ProjectB/
            Budget.xlsx
            Timeline.docx

Advantages

  • Directory structures allow grouping of related files and easier access.
  • File-based storage is flexible and simple but can become hard to manage with many files.

6.6.3 Know the definition of hierarchy-based structure and understand its purpose and when it is used

Definition

A hierarchy-based structure organizes data in a tree-like format where each parent node can have multiple child nodes.

Purpose and Usage

    Efficiently represents relationships between items.

    Supports quick navigation, retrieval, and reporting.

    Commonly used in file systems, hierarchical databases (XML, LDAP), and organizational charts.

Advantages

  • Easy to understand and visualize relationships.
  • Reduces data redundancy by sharing parent data across multiple child nodes.
  • Supports hierarchical queries, e.g., “Show all employees under IT department.”

Example

        Company
        ├── HR
        │    ├── Recruitment
        │    └── Payroll
        └── IT
            ├── Development
            └── Support

Comparison with Other Structures

  • File-based: Flat, simple, suitable for small datasets.
  • Directory-based: Nested folders, medium complexity, suitable for medium to large datasets.
  • Hierarchy-based: Tree-like, complex, suitable for complex relationships.

6.6.4 Understand the interrelationships between storage structures and data transformation

Definition

Data transformation is the process of converting data from one format or structure to another for easier analysis, storage, or compatibility.

Interrelationships with Storage Structures

    File-based: May require combining or splitting files for analysis. Example: Consolidating CSV files into a single dataset.

    Directory-based: Automated processing must navigate nested directories. Example: A program reads multiple project folders to generate reports.

    Hierarchy-based: Mapping relationships carefully when transforming into flat formats. Example: Converting XML or JSON hierarchical data into relational tables.

Role of Metadata

    Metadata guides transformation by providing context, definitions, and relationships, ensuring accurate interpretation.

Real-world Example

A company stores sales data in multiple folders by region (directory-based), and each file contains nested customer order details (hierarchy-based). To analyze total sales, files are aggregated and hierarchical data flattened for spreadsheet analysis, using metadata to maintain correct field mapping.
