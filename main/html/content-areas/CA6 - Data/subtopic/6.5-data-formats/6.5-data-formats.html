Subtopic 6.5: Data Formats

6.5.1 Know the definition of common data formats and understand their purpose and when each is used

JSON

    Definition: Lightweight, text-based format for storing and exchanging structured data.

    Purpose: Human-readable and machine-parsable; supports nested structures (objects and arrays).

    When used: Web APIs, configuration files, and data exchange between front-end and back-end systems.

    Example: { "name": "Alice", "age": 25, "isStudent": true, "subjects": ["Math", "English"] }

Text File (.txt)

    Definition: Simple files containing plain text without any structured formatting.

    Purpose: Useful for human-readable notes, logs, or small datasets.

    When used: Logging system events, storing configuration notes, or simple data storage.

    Example: ``` Name: Alice Age: 25 Status: Student ```

CSV (Comma-Separated Values)

    Definition: Text files storing tabular data, with values separated by commas (or other delimiters).

    Purpose: Simple, human-readable format for tabular datasets.

    When used: Exporting spreadsheets, importing/exporting databases, and dataset analysis.

    Example: ``` Name, Age, Status Alice, 25, Student Bob, 30, Teacher ```

UTF-8

    Definition: Character encoding standard capable of representing almost all characters in all languages.

    Purpose: Ensures correct display of text across different platforms and languages.

    When used: Websites, APIs, internationalized software, and text files.

    Example: `"你好"` (Chinese) and `"Hello"` (English) stored safely in UTF-8 files.

ASCII

    Definition: Character encoding standard for basic English letters, digits, and symbols (128 characters).

    Purpose: Lightweight encoding for simple text, widely compatible.

    When used: Legacy systems, simple text files, and communication protocols.

    Example: Letters `A-Z`, numbers `0-9`, and symbols like `@` or `#`.

XML (Extensible Markup Language)

    Definition: Markup language for encoding structured data in a human- and machine-readable way.

    Purpose: Supports hierarchical data with tags describing the data.

    When used: Configuration files, document storage, data exchange between enterprise systems, RSS feeds.

    Example: Alice 25 Student Math English

6.5.2 Understand the interrelationships between data format and data transformation and make judgements about the suitability of using data formats in digital software development

    Data transformation: The process of converting data from one format to another, such as JSON → CSV or XML → JSON.

    Interrelationship: Structured formats (JSON, CSV, XML) are easier to transform and validate due to predictable structures. Text files and encodings (UTF-8, ASCII) determine readability and compatibility during transformation.

    Judgements for suitability:

    • JSON: Ideal for web apps and APIs; supports hierarchical data and is easy to parse.
    • Text file: Lightweight and simple for logs or notes; not suitable for complex structured data.
    • CSV: Perfect for tabular datasets and spreadsheet compatibility; cannot handle nested data easily.
    • UTF-8: Essential for internationalized applications and multilingual content.
    • ASCII: Lightweight and compatible but limited to basic English characters.
    • XML: Suitable for hierarchical or complex data; more verbose than JSON but widely used in legacy systems.

    Key principle: Selecting the correct data format ensures efficient storage, smooth data exchange, and compatibility across systems. Developers should balance readability, structure, size, and interoperability when choosing a format.
